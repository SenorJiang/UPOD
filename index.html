Git-hub Rules:

  • The person who opens an issue, closes the issue. The exception to this is moderators and SQA team leaders.
  • Use the local resources. Don't use resources on other sites. If you need an external resource, ask me and i'll add a local copy to the repo.
  • Use the file structure in place. New folders are created by administrators, ask them if you need to make one.
  • Use descriptive commits. Explain what it is you did. "$ git commit -m "message" is a useful command.
  • If someone who opens an issue is unsure of how to resolve it, they may not be able to close it. This is perfectly fine and DBrown will like seeing our teamwork. I have added a #help-wanted label to our repo so if that is the case, then someone can step in and help them resolve it.
  • Speaking of labels, I have added several labels to Github: #front-end, #back-end,#requirements, #help-wanted and #bug. Please use these labels as the issues will soon be overfilled with an amalgamation of issues from various different parts of the application. Labelling your issues now will allow the team members to filter by their role. It will help everyone use Github more effectively!
