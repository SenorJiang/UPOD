1. Introduction

1.1 Scope

  • The physics group will provide the word-to-word content that will be placed onto the website.
  • Be easy to navigate, as students generally stop looking unless information is easily available
  • Be responsive, as students generally are on tight timelines, and don't have time for sites to load slowly.

    • 1.2 Definitions, Acronyms, and Abbreviations

  • Interactive Diagram: a diagram that can take input from any visitor on the website, and the resulting diagram will be changed accordingly.
  • Stationary Diagram: A diagram that is fixed, in the sense that it does not move, or change with or without user interaction (e.g, pictures).
  • Animated Diagram: A diagram that is constantly moving with or without user interaction.

    • 1.3 References

  • There will be no occurences of references to external sources as of yet.

    • 1.4 Overview

  • The physics requirements document will demonstrate how the research will be displayed on the screen (how the user will see it).
  • This document will also help understand the format in which the research will be given to each team that requires it.

    • 2. Overall Description

    2.1 Product Perspective

    The physics research itself is independent from the other teams. When dividing the research into certain objects, collaboration with the MathJax, Graphics, and Back-End teams will be required.

    2.1.1 User Interfaces

  • Users will only be interacting with the interactive animated diagrams.

    • 2.2 Constraints

  • The user (person who can access the website) has the possibility to misuse the interactive diagram, and produce bizarre results.
  • Divisions by zero in equations, finding the square root of a negative number (imaginary numbers).

    • 3. Specific Requirements

    3.1 External Interfaces

    Inputs and outputs from the software system are not applicable to the Physics team, as our sole job is to provide the written context and concepts.

    3.2 Organizing the Specific Requirements

    3.2.1 User Class

  • User can access all physics content on the website, including interactive diagrams
  • Admins can access and change all the physics content on the website, including necessary adjustments to all diagrams (interactive, animated or stationary).

    • 3.2.2 Objects

  • Physics research objects will be converted into the following:
  • -----> Plain text: general text for the user to see
  • -----> Equations: equations related to the topic/subtopic
  • -----> Constants: constants defined within the equation
  • -----> Diagrams: types of media for better understanding of concepts
  • -----FIX-----> Interactive
  • ----THESE----> Stationary
  • ---INDENTS---> Animated
